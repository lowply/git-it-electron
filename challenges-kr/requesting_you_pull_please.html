원본 Patchwork 저장소(repository)에 풀 리퀘스트(pull request)를 서밋(submit)해보세요.

풀 리퀘스트(pull request)

종종 포크한 어떤 프로젝트를 수정하고 발전시키고 나면, 원본 작성자에게 그 변경들을 보내고 그들이 원본에 그것들을 풀하도록 요청해서 모든 사람들이 그 혜택을 받을 수 있게 하고 싶을 것입니다. 그것이 풀 리퀘스트입니다.

우리는 당신의 이름이 워크샵 완주자 리스트에 추가되기를 원합니다. 당신의 이름 파일을 추가하기 위한 풀 리퀘스트 를 만들어주세요: github.com/jlord/patchwork.

풀 리퀘스트(pull request) 생성

GitHub 에서 포크한 원본 저장소를 방문하세요. 지금의 경우에는 http://github.com/jlord/patchwork 입니다.

포크한 저장소에 브랜치를 푸시한 경우, 대체로 GitHub 가 이를 감지하고 원본 저장소의 웹사이트 위쪽에 이를 표시해줍니다. 그렇게 당신의 'add-<username>' 브랜치가 보인다면, 그곳을 클릭하여 풀 리퀘스트를 생성할 수 있습니다. 만약 그렇지 않은 경우:

  • 'New pull request' 버튼을 클릭하세요.
  • 서밋하고 싶은 변경이 들어있는 브랜치를 선택하세요. 'add-<username>' 브랜치겠죠.

이제 서밋 과정에 있는 풀 리퀘스트에 대한 자세한 내용이 담긴 페이지를 보게 될 것입니다. 이 페이지는 당신의 풀 리퀘스트를 원본 저장소에 있는 'gh-pages' 브랜치와 비교하여 diff 형식으로 커밋과 변경을 보여줍니다.

원본 저장소가 기여 문서(contribution documentation)를 가지고 있다면, GitHub 가 링크를 보여줄 것입니다. 이것은 저장소 주인의 문서로, 그 프로젝트에 기여하기 위한 가장 좋은 방법을 알려줍니다-당신의 변경이 채택되기 위해서는 읽어보는 것이 매우 큰 도움이 됩니다!

페이지에 보이는 것들이 모두 문제 없어보인다면:

  • 초록색 'Create pull request' 버튼을 누르세요. 'Open a pull request' 페이지로 이동합니다.
  • 당신이 원본 작성자에게 풀하기를 제안하는 변경에 대한 제목과 설명을 추가하세요.
  • 초록색 'Create pull request' 버튼을 누르세요!

빙고! 풀 리퀘스트를 서밋했습니다—잠시 동안 지금 이 느낌을 실컷 누리세요.

당신의 풀 리퀘스트가 문제없다면, 몇 분 내로 머지될 것입니다. 만약 몇 분 내에 자동으로 머지되지 않는다면, 아마 @reporobot 으로부터 왜 머지되지 않았는지에 대한 코멘트를 받을 수 있을 것입니다. 그런 경우에는, GitHub 에서 당신의 풀 리퀘스트를 닫고, 필요한 변경들을 당신의 브랜치에서 수행한 다음, 그 변경들을 당신이 포크한 저장소에 푸시하고, 당신의 풀 리퀘스트를 다시 여세요(그러면 @reporobot 이 다시 확인할 것입니다).
