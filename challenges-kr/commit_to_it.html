당신의 새 저장소(repository)에 파일을 만들어보세요. Git을 이용하여 파일을 추가하고(add) 변경사항들을 커밋(commit)하세요.

커밋(commit)

커밋은 Git을 사용하는 데 있어 가장 중요합니다. 커밋들은 당신이 작업했던 것들을 저장하고 설명하는 순간들입니다. 커밋들은 당신의 프로젝트 이력의 체크표입니다.

새 파일 만들기

이제 당신은 저장소를 가지고 있습니다. 이 저장소에 파일을 추가해봅시다.

텍스트 편집기를 열고 새로운 빈 파일을 만드세요. "Hello!" 같은 텍스트를 적어보고 지난 챌린지에서 만들었던 'hello-world'폴더에 'readme.txt'라는 이름의 파일로 저장하세요.

상태(status) 확인하기 + 변경사항들을 추가하고(add) 커밋하기(commit)

다음으로 변경사항이 있는지 아닌지를 알아보기 위해 당신의 저장소의 상태 를 확인해봅시다. 당신은 어떤 것들이 바뀌었는지 알고 있지만 Git도 알고 있을까요?

이 명령어들을 실행할 때 'hello-world' 폴더 내에서 실행하고 있는지 확인하세요. 당신의 저장소에서 변경된 것들을 보기 위해 Git을 사용해봅시다:

먼저, 상태를 확인하세요:

git status

Git은 당신에게 추가된 파일을 알려줄 것입니다.

그 다음에 Git을 사용하여 커밋 (aka save)할 변경사항들의 일부로 만들기 위해, 방금 만든 파일을 추가 해주세요:

git add readme.txt

마지막으로, 업데이트된 내용에 대해 설명한 짧은 메시지를 포함하여 변경사항들을 저장소의 이력에 커밋하세요:

git commit -m "Created readme"

더 많은 변경사항 만들기

'readme.txt'에 다른 라인을 추가하고 파일을 다시 저장하세요.

터미널에서 지금의 파일과 마지막 커밋 간의 차이점(diff) 을 볼 수 있습니다.

Git에서 차이점(diff) 보기:

git diff

이제 지금 배운 것들들을 이용하여 최종 변경사항을 커밋해보세요.

Please select your repository directory.
  • 저장소에서 변경사항의 상태 확인하기
  • git status

  • git diff
  • 커밋하려는 파일의 변경사항을 추가하기
  • git add <파일이름>
  • 모든 파일들의 변경사항을 추가하기
  • git add .
  • 변경사항을 설명한 짧은 메시지를 포함하여 변경사항을 커밋하기(aka save)
  • git commit -m "<커밋 메시지>"
