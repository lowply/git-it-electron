Pull Requests

Quando você faz mudanças e melhorias para um projeto que você forked, frequentemente você vai querer enviar essas alterações para o maintainer do original e solicitar que eles extraim as alterações do original, de modo que todos possam beneficiar das atualizações em que é um pull request.

Queremos adicionar você à lista de finalistas da oficina, então faça uma pull request para o original: github.com/jlord/patchwork.

Crie uma pull request

Visite o repositório original que você forked no GitHub, neste caso http://github.com/jlord/patchwork.

Frequentemente GitHub vai detectar se você empurrou um branch para um fork e exibe no topo do site original.Se você ver que com o seu 'add-username' branch, você pode clicar para criar uma Pull Request de lá. Se não:

  • Clique no butão verde 'New pull request' .
  • Selecione o branch com as mudanças que deseja enviar. Ele deve ser o único com 'add-yourusername'.

Você vai ver agora uma página com os detalhes do pull request você está no processo de apresentação. Esta página mostra os commits e mudanças, sob a forma de um diff, associado à sua pull request em comparação com 'gh-pages' branch do original.

Se o repositório original tem uma documentação de contribuição, GitHub vai ligar a isso. Esta é a documentação dos proprietários de repositórios de como fazer contribuições melhores para esse projeto—muito útil para ler se você quiser ver suas alterações adotadas!

Se tudo na página parece bom e como você espera:

  • Clique 'Criar pull request'
  • Adicione um título e uma descrição das alterações que você está segerindo ao autor original adiciona-las.
  • Clique 'Enviar pull request'!

Toca aqui! Você enviou o seu pull request, demorar alguns segundos para aquecer no momento..

Se está tudo bem com o seu pull request, ele vai ser fundido dentro de momentos. Se ele não fundir automaticamente em alguns momentos, então você vai provavelmente ter alguns comentários de Reporobot sobre por que não poderia fundir. Se este for o caso, feche o seu pull request no GitHub, faça as mudanças necessárias para o seu branch, empurre essas alterações de volta para o seu fork e reenvie o seu pull request.
